# 企业浏览器测试矩阵

> 页面内置数据均为程序合成的 MOCK、不来源于真实 KYC；用户新增或编辑的内容仅为“用户确认的测试数据”，页面不验证其真实性。站点无后端、仅在本机浏览器处理。网页显示的“完成/允许/拒绝”只代表前端 API 结果，不证明企业浏览器管理后台已形成告警、审计或阻断记录。

> 安全 fixture 边界：身份证使用不存在的 `990000` 行政区码并计算正确校验位，严格行政区校验可能不命中；手机号使用不可拨打的 `000-TEST-xxxx` 模式。若规则必须覆盖真实行政区、号段或账户格式，请在受控环境使用组织批准的测试向量。禁止联系、拨打、支付或核验页面标识符。

| 类别 | 测试项 | 页面触发方式 | 期望企业浏览器控制点 | 网页能力边界 |
|---|---|---|---|---|
| 内容识别 | 身份证格式与校验位 | 客户档案 → 临时显示敏感字段 | 仅格式/校验位规则、显示水印、访问审计 | 行政区码 `990000` 不存在；严格行政区规则可能不命中 |
| 内容识别 | 护照测试标识 | 客户详情 → 临时显示 | 证件类关键字或组合识别 | 使用 `MOCK-PASS-xxx`，严格真实护照格式规则可能不命中 |
| 内容识别 | 手机测试标识与邮箱 | 客户列表/详情 | 联系方式关键字、页面访问审计 | 手机为不可拨打模式；`.test` 邮箱不可用于真实邮件投递 |
| 内容识别 | 银行卡、IBAN、SWIFT | 客户详情 | 金融账号分类、显示或外发限制 | 银行卡为常用行业测试号，IBAN 为公开示例；禁止支付或核验 |
| 内容识别 | 地址、公司、职业组合 | 客户详情或组合 KYC | 多字段组合识别与风险升级 | 同上 |
| 页面访问 | MOCK KYC 页面访问 | 打开站点或切换模块 | URL 分类、访问策略、访问日志 | 网页无法读取浏览器管理策略 |
| 展示控制 | 默认脱敏 | 打开客户列表 | 浏览器策略与应用脱敏叠加验证 | 页面自身实现脱敏，不代表浏览器做了脱敏 |
| 展示控制 | 临时显示明文 | 单击“临时显示敏感字段” | 敏感信息展示告警、水印或阻断 | 页面仅记录非敏感操作元数据 |
| 复制 | 脱敏单个证件号码 | 复制与粘贴 → 脱敏单字段 | 剪贴板 DLP、复制阻断或审计 | 默认复制已脱敏；页面只能获知写入 API 成功/失败 |
| 复制 | 脱敏组合 KYC 文本 | 复制与粘贴 → 脱敏组合 KYC | 多字段内容识别、风险升级 | 姓名及敏感字段默认脱敏 |
| 复制 | 脱敏 JSON | 复制与粘贴 → 脱敏 JSON | 结构化内容识别 | 同上 |
| 复制 | 脱敏 Base64 | 复制与粘贴 → 脱敏 Base64 | 编码内容检测或阻断 | 页面不验证浏览器是否解码识别 |
| 复制 | 未脱敏组合 KYC | 发起未脱敏复制 → 阅读摘要 → 输入确认词 | 高风险剪贴板 DLP、阻断或审计 | 二次确认只属测试护栏；完成后立即覆盖剪贴板，页面从不读取剪贴板 |
| 粘贴 | 手动粘贴 | 在粘贴区使用键盘/菜单 | 入站粘贴限制、跨域数据流控制 | 页面不调用 clipboard.read，内容刷新即丢失 |
| 下载 | 脱敏 CSV | 导出与下载 → CSV | 下载审计、文件内容检查 | 下载由 Blob 在本机生成；公式前缀会被安全转义 |
| 下载 | 脱敏 JSON | 导出与下载 → JSON | 结构化文件 DLP | 同上 |
| 下载 | 脱敏 TXT | 导出与下载 → TXT | 文本文件 DLP | 同上 |
| 下载 | 脱敏 HTML | 导出与下载 → HTML | 主动内容文件下载策略 | 页面不会自动打开下载文件 |
| 下载 | 脱敏 XML | 导出与下载 → XML | 标签化内容检测 | 下载由 Blob 在本机生成 |
| 下载 | 脱敏 PDF | 导出与下载 → PDF | PDF 下载、文件审计、水印策略 | PDF 动态内容规范化为 ASCII，测试会校验 xref；网页不能判断下载后控制 |
| 下载 | MOCK 证件 PNG | 导出与下载 → 证件 PNG | OCR/图像 DLP、图片下载策略 | Canvas 生成；无绘图上下文或编码返回空值时明确报失败，不记录成功 |
| 下载 | 未脱敏 CSV | 未脱敏导出 → 阅读摘要 → 输入确认词 | 高风险内容阻断、审批、下载审计 | 页面确认词只是测试护栏，不是可信授权机制 |
| 下载 | 普通 Blob | 其他下载行为 → Blob | 通用下载策略和文件落盘控制 | 页面只能触发下载 |
| 上传 | 文件选择 | 文件与打印 → 选择文件 | 文件上传入口限制、扩展名策略 | 页面只取 name/type/size/lastModified，不读取内容、不上传 |
| 上传 | 拖放文件 | 拖到上传区域 | 拖放通道控制 | 同上 |

| 导航 | 同源弹窗 | 打开同源测试窗口 | 弹窗策略、新窗口审计 | 使用随机窗口名并清除 `opener`；只能判断 window.open 是否返回窗口对象 |
| 导航 | 外部链接 | 打开 example.com | 外联限制、URL 分类、隔离打开 | 无法确认浏览器后台策略原因 |
| 页面防护 | 静态 CSP 与 Referrer Policy | 打开页面并检查请求/导航 | 同源资源策略、无网络连接、来源信息控制 | Meta CSP 允许同源资源及本机 `data:`/`blob:` 图片；服务端响应头仍需由托管平台配置 |
| 权限 | 通知 | 浏览器权限 → 请求通知 | 通知权限默认值、用户授权策略 | 不发送通知或后台推送 |

| 权限 | 摄像头 | 浏览器权限 → 请求摄像头 | 媒体权限禁用、提示或审计 | 授权后立即 stop 所有轨道，不预览/录制 |
| 权限 | 麦克风 | 浏览器权限 → 请求麦克风 | 媒体权限禁用、提示或审计 | 授权后立即 stop 所有轨道，不监听/录音 |

## 建议的执行顺序

1. 使用隔离测试账号打开页面，确认顶部持续显示“MOCK 模拟数据 / 无后端 / 仅本机处理”，并阅读安全 fixture 边界。
2. 先记录企业浏览器当前策略版本、设备、账号、时间和预期结果。
3. 依次执行展示、复制、下载、上传入口、打印、权限、新窗口测试。
4. 每执行一项，立即在企业浏览器管理后台核对是否产生预期阻断或审计。
5. 最后执行人工清单，并保存企业浏览器侧的证据；不要把网页会话日志当作控制有效性证据。
6. 未脱敏复制测试后立即覆盖剪贴板；全部测试完成后清空会话日志，并按组织要求清除该站点 localStorage 或使用“恢复初始数据”。

## 判定原则

- **网页显示成功**：只说明浏览器 API 调用未抛错或下载被触发。
- **浏览器控制有效**：必须在用户侧看到预期阻断/提示/水印，并在需要时由管理后台日志佐证。
- **未知/不适用**：浏览器不支持 API、非 HTTPS 环境、操作没有用户手势或组织策略未覆盖时，应记录为未知/不适用，而不是通过。
