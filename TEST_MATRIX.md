# 企业浏览器测试矩阵

> 页面内置数据均为程序合成的 MOCK、不来源于真实 KYC；用户新增或编辑的内容仅为“用户确认的测试数据”，页面不验证其真实性。站点无后端、仅在本机浏览器处理。网页显示的“完成/允许/拒绝”只代表前端 API 结果，不证明企业浏览器管理后台已形成告警、审计或阻断记录。

> 高拟真 fixture 边界：48 条记录覆盖 16 个国家和地区，默认完整展示姓名、身份标识、护照、手机号、账户和地址。中国身份证使用真实行政区格式、顺序码 `000` 和正确校验位；美国/英国等使用公开测试段或合成格式。它们用于内容、模式和 OCR 检测，不保证通过每个国家的生产级业务校验。禁止联系、拨打、支付或核验页面标识符。

## 建议的执行顺序

1. 使用隔离测试账号打开页面，确认顶部持续显示“MOCK 模拟数据 / 无后端 / 仅本机处理”，并阅读安全 fixture 边界。
2. 先记录企业浏览器当前策略版本、设备、账号、时间和预期结果。
3. 依次执行展示、复制、下载、上传入口、打印、权限、新窗口测试。
4. 每执行一项，立即在企业浏览器管理后台核对是否产生预期阻断或审计。
5. 最后执行人工清单，并保存企业浏览器侧的证据；不要把网页会话日志当作控制有效性证据。
6. 未脱敏复制测试后立即覆盖剪贴板；全部测试完成后清空会话日志，并按组织要求清除该站点 localStorage 或使用“恢复初始数据”。

## 判定原则

- **网页显示成功**：只说明浏览器 API 调用未抛错或下载被触发。
- **浏览器控制有效**：必须在用户侧看到预期阻断/提示/水印，并在需要时由管理后台日志佐证。
- **未知/不适用**：浏览器不支持 API、非 HTTPS 环境、操作没有用户手势或组织策略未覆盖时，应记录为未知/不适用，而不是通过。
