# 反洗钱演示系统｜企业浏览器安全能力测试靶场

这是一个部署在 GitHub Pages 上的纯静态测试页面，用于验证企业浏览器的内容识别、数据防泄漏、下载限制、权限控制和审计能力。

## 重要边界

- 页面内置记录由程序合成，不来源于真实 KYC；采用高拟真的姓名、地址、证件、护照、手机号和金融标识格式，默认完整显示，专门用于企业浏览器 DLP、OCR、复制、下载、打印和截图策略测试。
- 覆盖中国、美国、英国、德国、法国、新加坡、阿联酋、土耳其、印度、日本、韩国、巴西、墨西哥、加拿大、澳大利亚和巴拿马共 16 个国家和地区。
- 中国身份证的出生日期与档案一致并计算 GB 11643 校验位；法国 NIR、新加坡 NRIC、土耳其 TCKN、韩国 RRN、巴西 CPF、加拿大 SIN 和澳大利亚 TFN 生成校验位。美国 SSN 使用 `900` 测试段，英国 NINO 使用 `QQ` 测试前缀，美国/加拿大手机号使用 `555-01xx` 示例段。严格生产级名录校验不一定命中，但格式、内容和 OCR 规则可识别完整载荷。
- 银行卡 PAN 使用公开支付沙箱测试向量，卡组织/IIN 与字段标签一致且逐条通过 Luhn；有效期为未来日期，CVC 仅为沙箱测试值（真实生产系统禁止留存 CVC）。
- 英国、德国、法国、阿联酋、土耳其和巴西使用符合各国长度/BBAN 结构且通过 MOD-97 的 IBAN；不采用 IBAN 的国家明确标记为不适用，并提供对应本地清算代码与合成账户号。BIC/SWIFT 使用国家码一致、不可路由且不声称已注册的测试格式。
- 护照记录含 44 字符 ICAO 9303 风格 MRZ，两行中的证件号、国籍、出生日期、性别、到期日及组合校验位相互一致。
- 没有后端、数据库、登录服务或数据上报接口。
- KYC 测试记录只保存在当前浏览器的版本化 localStorage key：`aml-browser-lab:kyc:v5`；旧版本数据不会被继续复用。
- 若企业策略禁用 localStorage/sessionStorage，页面自动降级为“内存模式”，CRUD 与日志仍可测试，但刷新即丢失；页头会持续提示该状态。
- 新增或编辑前必须确认仅使用合成测试信息。用户输入内容由页面标记为“用户确认的测试数据”，页面不验证或保证其真实性。
- 粘贴区内容不持久化；上传/拖放功能只查看浏览器提供的文件元数据，不读取文件内容。
- 摄像头和麦克风一旦授权会立即停止全部媒体轨道，不预览、不录制、不上传。
- 媒体轨道清理位于 `finally`，即授权后的状态处理异常也会尝试停止全部轨道。
- 地理位置测试只显示允许/拒绝结果，不显示或保存坐标。
- 页面不读取剪贴板，只能在用户单击后写入测试内容。默认复制完整高拟真 MOCK 字段；另保留确认词场景和覆盖剪贴板按钮，便于分别测试直接复制与高风险确认流程。
- CSV 导出会对以 `= + - @` 开头（含前导空白）的单元格加安全前缀，降低电子表格公式注入风险。
- 会话日志只记录测试时间、测试类型和结果，不记录原始 PII、坐标或文件名。
- 页面使用纯静态 CSP：只加载同源脚本、样式和字体，禁止网络连接及嵌入内容；仅为本机生成的图片/下载保留 `data:` 与 `blob:` 图片、媒体能力，并设置 `no-referrer`。
- 同源测试弹窗使用随机窗口名并在写入内容前清除 `opener`，避免窗口复用和反向控制父页面。
- 截图、水印、页面另存为、开发者工具和浏览器管理后台审计结果无法由网页可信检测，必须人工验证。

## 包含的测试面

1. 48 条、16 个国家和地区的高拟真 MOCK KYC 记录，支持搜索、风险筛选、分页、选择、详情、新增、编辑、软删除、撤销与重置。
2. 姓名、身份标识、护照/MRZ、手机号、邮箱、银行卡 PAN/卡组织/有效期/测试 CVC、本地账户、IBAN、SWIFT 和地址默认完整显示；可切换脱敏预览作为对照。
3. 含完整字段的 CSV、JSON、TXT、HTML、XML、PDF、身份证件 PNG 和护照 PNG 下载。
4. 完整 CSV 的直接下载和确认词高风险下载两种测试路径。
5. 默认完整的单字段、组合 KYC、JSON、Base64 剪贴板写入；确认词保护的组合复制；剪贴板覆盖按钮及手动粘贴区。
6. 文件选择/拖放元数据、打印、新窗口、外部链接和普通 Blob 下载。
7. 通知、位置、摄像头、麦克风、剪贴板写入、弹窗、全屏权限测试。
8. 详情抽屉内直接渲染含完整字段、人像占位图和 MOCK 水印的身份证件与护照 Canvas 图片；另提供字段完全相同的 Canvas 与 `<img src>` 静态图片对照组。全部图片均可点击放大、按 Esc 关闭并下载，用于对比企业浏览器对不同渲染方式的 OCR、截图、水印和图片下载策略。
9. 人工验证清单和无 PII 会话日志导出。

完整的控制点、触发方式和网页能力边界见 [TEST_MATRIX.md](./TEST_MATRIX.md)。

## 规则依据

- 支付卡测试 PAN、卡组织、未来有效期和 CVC 长度：[Stripe Test card numbers](https://docs.stripe.com/testing)
- IBAN 国家结构与长度：[Swift ISO 13616 IBAN Registry](https://www.swift.com/standards/data-standards/iban-international-bank-account-number)
- BIC 结构：[Swift Business Identifier Code](https://www.swift.com/standards/data-standards/bic-business-identifier-code)
- 真实生产环境不得在授权后留存 CVC/CVV/CID：[PCI SSC FAQ 1210](https://www.pcisecuritystandards.org/faqs/1210/)

## 本地运行

该页面不需要安装依赖或构建：

```bash
python3 -m http.server 8080
```

然后访问 `http://127.0.0.1:8080/`。通知、媒体、地理位置、剪贴板和全屏等 API 受浏览器安全上下文、用户手势及企业策略影响；GitHub Pages 的 HTTPS 环境更接近正式测试条件。

## 发布

仓库根目录就是可发布内容，GitHub Pages 可直接从默认分支根目录提供服务。发布前至少执行：

```bash
node --check assets/app.js
node tests/security.test.mjs
node tests/dom-contract.test.mjs
```

## 测试安全提示

不要把真实个人信息粘贴、录入或替换到此测试站点，也不要拨打、联系、支付或核验任何测试向量。若规则必须匹配真实行政区、手机号段或账户格式，请只在受控环境使用组织批准的测试标识符和隔离测试账号。
